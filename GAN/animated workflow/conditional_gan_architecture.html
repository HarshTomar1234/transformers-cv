Conditional GAN Architecture

Generative Adversarial Network with Class Conditioning
Mirza & Osindero, 2014 — arXiv:1411.1784
Noise Input z
Condition y
Generator G

Loss / Output

⬡ Generator G(z, y)

Takes a random noise vector z concatenated with class label y and maps it to a data sample. The condition guides generation toward a specific class (e.g., a particular digit in MNIST).

G : (z, y) → x_fake
z ~ N(0, I), y = one-hot(class)

◈ Discriminator D(x, y)

Receives either a real sample x or fake sample G(z,y), concatenated with the same condition y, and classifies whether the input is real or generated.

D : (x, y) → [0, 1]
1 = real, 0 = fake

◆ Conditioning Mechanism

The condition y (class label, attribute, text embedding) is the key differentiator from vanilla GAN. It is concatenated as additional input to both networks, enabling controlled, targeted generation.

Vanilla GAN: G(z), D(x)
cGAN: G(z,y), D(x,y)

▲ Objective Function

A minimax game conditioned on y. The discriminator maximizes its ability to distinguish real from fake, while the generator minimizes the discriminator's success — both conditioned on the same label.

min_G max_D V(D,G) =
E_x[log D(x|y)] + E_z[log(1 − D(G(z|y), y))]
