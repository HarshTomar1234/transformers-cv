⬡ Generator — Transposed Convolutions

Upsamples the latent vector z through fractional-strided (transposed) convolutions. Each layer doubles spatial resolution while halving channels. Uses BatchNorm + ReLU for all layers except the output (Tanh).

z (100×1×1) → 1024×4×4 → 512×8×8 → 256×16×16 → 128×32×32 → 3×64×64

◈ Discriminator — Strided Convolutions

Mirror architecture of G. Uses strided convolutions (stride=2) instead of pooling to downsample. LeakyReLU (α=0.2) for all layers. No BatchNorm in the first layer. Final Conv outputs a single scalar via Sigmoid.

3×64×64 → 64×32×32 → 128×16×16 → 256×8×8 → 512×4×4 → 1 (sigmoid)

◆ vs Vanilla GAN

Replaces all fully-connected layers with convolutional layers. Removes all pooling layers. Adds BatchNorm to stabilize training. These changes enable generation of spatially coherent images instead of noisy pixel arrays.

Vanilla: FC only, unstable, blurry
DCGAN: All-Conv, stable, sharp 64×64 images

▸ DCGAN Architectural Guidelines

Rule 1

Replace all pooling layers with strided convolutions (discriminator) and fractional-strided convolutions (generator)

Rule 2

Use BatchNorm in both G and D — except G's output layer and D's input layer

Rule 3

Remove all fully-connected hidden layers for deeper architectures

Rule 4

Use ReLU in Generator (all layers except output=Tanh), LeakyReLU in Discriminator (all layers)
