Progressive Training: Both G and D Grow Simultaneously

⬡ Progressive Growing

Both G and D start at 4×4 and grow symmetrically to 1024×1024. New layers added one resolution at a time, with training stabilizing at each level before growing. This lets the network learn coarse structure first, then progressively refine details.

4×4 → 8×8 → 16 → 32 → 64 → 128 → 256 → 512 → 1024
Each level: stabilize → fade-in → stabilize

◆ Smooth Fade-in (α)

When a new layer is added, it's blended in gradually using parameter α (0→1). Output = (1−α)·old_path + α·new_path. This prevents sudden training instability from adding untrained layers. Both G and D use matching fade-in.

α linearly increases 0→1 during transition
Prevents shock from untrained new layers
Applied symmetrically in G and D

◈ Training Stabilization Tricks

Uses WGAN-GP loss, Pixel Normalization (instead of BatchNorm), Equalized Learning Rate (scale weights at runtime), and Minibatch Standard Deviation in D (fights mode collapse). These combined enable stable 1024×1024 generation.

Loss: WGAN-GP (gradient penalty)
Norm: PixelNorm (per-pixel L2 norm)
LR: Equalized (He init at runtime)

★ Why Progressive?

Training 1024×1024 from scratch is extremely unstable. By growing progressively: the network learns large-scale structure at low resolution first, training is faster (most time at low-res), and each resolution level is well-initialized by the previous one.

Stable: gradual complexity increase
Fast: low-res training is cheap
Quality: first GAN to generate 1024×1024 faces

▸ Training Stabilization Techniques

Equalized Learning Rate

Instead of careful weight initialization, weights are initialized N(0,1) and scaled by the He constant at runtime. Ensures all parameters learn at the same speed regardless of layer shape.

w_runtime = w · √(2/fan_in)
Scaling applied dynamically each forward pass
Pixel Normalization

Replaces BatchNorm in G. Normalizes each pixel's feature vector to unit length. Prevents signal magnitudes from escalating during training — simpler and more stable than BN.

PixNorm(x) = x / √(mean(x²) + ε)
Applied per-pixel across channels
Minibatch Std Deviation

Appended to D's last layer. Computes std deviation across the batch for each spatial location, averages to one value, and replicates as an extra feature map. Encourages diversity — fights mode collapse.

σ = stddev(features across batch)
Appended as extra channel in D
WGAN-GP Loss

Uses Wasserstein distance with gradient penalty instead of standard GAN loss. Provides stable gradients at all resolutions and correlates with visual quality — essential for progressive training to work.

L = E[D(fake)] − E[D(real)] + λ·GP
λ = 10, no weight clipping
