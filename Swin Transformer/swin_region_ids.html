Region IDs & Masking in Swin Transformer

How Swin uses cyclic shift + region tracking + masking to compute shifted window attention efficiently

1

Regular Window Partition (Layer ℓ)

We divide the 8×8 feature map into 4 non-overlapping 4×4 windows. Each window computes self-attention independently. Every cell shows its Window ID.

How it works

All tokens in Window A attend to each other. All tokens in Window B attend to each other. No cross-window attention. Simple, efficient, but creates hard boundaries — edge tokens can't see their neighbors in adjacent windows.

2

Shifted Window Partition (Layer ℓ+1)

We shift the window grid by (M/2, M/2) = (2, 2). Now the window boundaries fall at different positions, creating 9 sub-regions of irregular sizes. Each sub-region gets a unique Region ID.

After Shifting → 9 Irregular Sub-Regions with Region IDs

Region H (2×4)
Region I (2×2)
Problem: 9 windows of different sizes

GPUs hate irregular sizes. We can't batch 9 differently-shaped attention computations efficiently. We need to get back to 4 uniform 4×4 windows.

🔑 Role of Region IDs

Each of the 9 sub-regions gets a unique ID (A through I). This ID remembers the original spatial location of each token. When we rearrange tokens via cyclic shift, the Region ID travels with the token. Later, we use these IDs to determine: should these two tokens attend to each other or not?

3

Cyclic Shift (torch.roll)

We cyclically shift the feature map by (-2, -2): top 2 rows wrap to the bottom, left 2 columns wrap to the right. The Region IDs stay attached to their tokens. Now we partition into 4 regular 4×4 windows again.

What Region IDs tell us

Window 0 (top-left): All tokens have Region ID = E → pure window, no mask needed.
Window 1 (top-right): Mix of Region E + Region D → needs masking!
Window 2 (bottom-left): Mix of Region E + Region B → needs masking!
Window 3 (bottom-right): Mix of Regions E + D + B + A → needs masking!

Region IDs are the mechanism that tells the mask which pairs to block.

4

Mask Generation Using Region IDs

For each of the 4 windows, we generate a mask matrix. The rule is dead simple:

If Region_ID(token_i) == Region_ID(token_j) → mask = 0 (allow)
If Region_ID(token_i) != Region_ID(token_j) → mask = −∞ (block)

Let's see this for Window 3 (bottom-right) — the most complex case with 4 mixed regions:

mask = −∞ (different Region ID → blocked)
Reading the mask matrix

The block-diagonal green pattern shows that tokens from the same region only attend to each other. The off-diagonal red blocks show all cross-region pairs are blocked. This is entirely determined by comparing Region IDs.

5

Putting It All Together

1
Regular Window Attention (Layer ℓ)

4 clean windows, each 4×4. Standard self-attention within each. No masks needed.

2
Shift grid by (M/2, M/2)

Creates 9 irregular regions. Assign Region IDs (A–I) to track origin.

3
Cyclic shift (torch.roll)

Wraps border fragments → back to 4 uniform 4×4 windows. Region IDs travel with tokens.

4
Generate mask using Region IDs

Same Region ID → mask=0 (attend). Different Region ID → mask=−∞ (block).

5
Compute masked attention

QKT/√d + bij + maskij → softmax → × Values. Masked pairs get exactly 0 weight.

6
Reverse cyclic shift

torch.roll by (+M/2, +M/2) → tokens back to original positions. Done!

The Genius of This Design

Region IDs are the bookkeeping system that makes the entire trick work. They track which tokens are genuine shifted-window neighbors vs. which were artificially grouped by cyclic shift. Without Region IDs, we couldn't generate the mask, and without the mask, cyclic shift would create false attention connections that corrupt the model.
